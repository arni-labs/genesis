## 7. Track A — Genesis GitHub workflow layer v1

**RFC first**: `docs/rfc/0004-github-workflow-layer.md` (outsider-readable, the whole slice).
**ADRs**: merge strategies + conflict semantics; push-path auth + force classification.
All new WASM follows the hard rules (WASM-only handlers, Cedar on every mutation, TigerStyle,
70-line functions, byte-exact serialization verified against real `git`).

### A1. Push auth + force-push classification (S) — first, so every endpoint is born gated

- **Where**: `wasm/git_receive_pack/src/lib.rs` (currently zero auth), reuse the token
  resolver from `wasm/git_refs_advertise/src/auth.rs` (SHA-256 token lookup over
  `/tdata/GitTokens`, scopes → `X-Temper-Principal-Scopes`); `wasm/scm_ingest_pack/src/lib.rs:270-283`.
- **How**: parse Authorization in receive-pack; reject anonymous push; commit-DAG ancestry
  walk in ingest so non-fast-forward updates dispatch `Ref.ForceUpdate` (Cedar `force`
  scope, `policies/ref.cedar:63-72`) instead of plain `Update`; fire `GitToken.MarkUsed`;
  tighten upload-pack anonymous fallback (`git_upload_pack/src/lib.rs:69-77`) per RFC
  decision; smokes stop registering push `RequiresAuth:false`.
- **Tests**: unit (ancestry classification, header parsing); round-trip: authed push OK,
  no/bad token 401/403, non-FF without `force` scope rejected with clear stderr, with scope allowed.

### A2. REST: repos + refs/branches (M) — `wasm/github_rest_repos`, `wasm/github_rest_refs`

- **Endpoints**: `POST /api/v3/user/repos` (create + auto-provision — closes RFC-0002
  Slice C and replaces the missing `repository_provision` workaround), `GET /repos/{owner}/{repo}`,
  `GET/POST/DELETE /repos/{o}/{r}/git/refs[/...]`, `GET /repos/{o}/{r}/branches`.
- **How**: kernel HttpEndpoint action bridge (ADR-0069 machinery, proven by
  `git_receive_pack/src/lib.rs:8-13`) mapping onto Repository/Ref entity actions; wire
  `Repository.CreateBranch` (currently inert spec) to a real effect.
- **Tests**: shape tests per hard rule 7 — recorded github.com response fixtures compared
  structurally (field names, types, required-presence) with the same call against Genesis.

### A3. REST: pulls + PR lifecycle (M) — `wasm/github_rest_pulls`, `wasm/scm_assign_pr_number`

- **Endpoints**: `POST/GET/PATCH /repos/{o}/{r}/pulls[/{n}]`, `POST .../pulls/{n}/reviews`,
  `PUT .../pulls/{n}/merge` (delegates to A4).
- **How**: map onto the existing dead state machines — `PullRequest.Open/UpdateHead/
  RequestReview/Approve/RequestChanges/Close` + `Review.Create` (`specs/pull_request.ioa.toml:24-97`);
  implement `scm_assign_pr_number` (spec already triggers it on Open: lines 126-132;
  max(Number)+1 over `/tdata/PullRequests`). Cedar becomes live: author-can't-approve
  (`policies/pull_request.cedar:77-86`), merge only from `Approved` (spec line 85).
- **Tests**: shape tests; state-machine tests (open→review→approve→merged; author
  self-approve denied; merge from non-Approved rejected).

### A4. Server-side merge (L) — `wasm/scm_merge_pr`

- **How**: Composite producer (same machinery as `Repository.IngestPack`) that: computes
  merge-base via paged commit-DAG walk over `/tdata/Commits`; executes strategy —
  **FF** (ref CAS advance only), **squash** (new single-parent commit reusing
  `commit_canonical_bytes`/`tree_canonical_bytes` from `crates/git_object` — two-parent
  merge commit already hash-tested at `commit.rs:139-158`), **clean merge** (two-parent
  commit when trees apply cleanly); divergent-content conflict → **409** with instructive
  body; emits Commit/Tree sub-writes using `scm_ingest_pack::build_object_row` row shapes
  (`lib.rs:529-600`) + CAS Ref advance + `PullRequest.Merge` transition; writes the new
  objects into the ADR-0011 raw-object cache so the next clone is warm.
- **Tests**: byte-equality against real `git` (perform the identical merge with git
  locally, compare commit/tree SHAs); conflict-409 case; each strategy round-trip;
  `git fsck` on a post-merge clone. This is the riskiest code in the plan — review depth
  scales accordingly.

### A5. CI round-trip gate (M)

- Promote the live smoke pattern into CI (`.github/workflows/ci.yml`): boot temper, mint
  two scoped GitTokens, then real `git`/`gh`: clone → branch → commit → push →
  `gh pr create` → `gh pr review --approve` (second token) → `gh pr merge` (matrix over
  FF/squash/merge) → re-clone → `git log` shows merge commit → `git fsck` → force-push
  denial case → REST shape suite. PRs that break it don't merge.

**Track A DoD**: A1–A5 merged in one genesis PR; CI gate green; live local run of the full
flow recorded; deployed to Railway and the same flow executed against production with
results captured; parity matrix rows flipped with test links; residual risks listed.
